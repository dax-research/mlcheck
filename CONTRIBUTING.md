# Contributing to mlcheck

Thanks for your interest in contributing! Please follow these guidelines:

- Fork the repository and create a feature branch.
- Run tests locally before opening a PR: `pytest`.
- Write clear, focused commits and a descriptive PR message.
- Follow existing code style and add tests for new functionality.
- For large changes, open an issue first to discuss the design.

Maintainers will review PRs and request changes as needed.
